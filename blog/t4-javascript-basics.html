Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'

Mozilla Web Docs defines control flow as, "the order in which the computer executes statements in a script." Simply put, the computer will do what the code tells it to do in the exact order in which the code is laid out.

The same Web Docs describes loops as, "a sequence of instructions that is continually repeated until a certain condition is met."

Using these definitions, we can apply the principles to the everyday analogy at a micro and macro level.

Micro: Every morning, my body checks that I have had enough sleep. If I have had enough sleep, I will wake up. If not, I will continue to sleep. If I woke up, I will go brush my teeth. I will brush my teeth until I feel like they are clean.

Macro: Every day, I will check my retirement fund. If I have met my goal (of having enough money to use for the rest of my life without working), I will not go to work. If I haven't met my goal with my retirement fund, I will go to work.

In both cases, what I am doing is performing a set of functions in a precise order. Within those functions, I will push through arguments (values that represent current conditions) against the statements, and I will keep looping in those functions until they are resolved.

Explain the difference between accessing data from arrays and objects

An array holds different pieces of information, and assigns them values indexing from 0. Think of it like an ordered list:

Colours:

  1. Red
  2. Blue
  3. Yellow

If I said, "Please choose #2 of the Colours list," the response I will expect is, "Yellow." With indexing from 0, Red is 0, Blue is 1, and Yellow is 2.

On the other hand, an object can hold values in an unlisted way. Using the Colours list above:

  • Red
  • Blue
  • Yellow

Here, I would have to ask for the word, "Yellow" if I wanted to use the value, "Yellow".

These are all rather simplistic terms, and I acknowledge I have a long way to go before being able to explain why a developer would use one over the other and for what use cases.

Explain what functions are, and why they are useful

These two words are going to be every developer's best friend: reusable code. Functions are code chunks that have performed a specific task and can be repurposed or re-used in either other codebases or contexts. They can even handle different use cases depending on the arguments that are pushed through them. Functions are likened to procedures or subroutines in other languages. They are a repeatable, repurposable task that can be called by other scripts or even other functions.
